Creare import dati provvisori YAML -> JSON DB

- Aggiungere una cartella locale per dati provvisori di development, per esempio `config/local_data/` o `db/local_data/`.
- Mettere la cartella in `.gitignore`, cosi i file YAML locali non vanno in repository.
- Creare un model per contenere dati importati in forma JSON.
- Nome consigliato: `Dataset` oppure `DataEntry`, evitando `Data` perche troppo generico.
- Campi iniziali:
  - `name`
  - `payload:json`
- Creare un task Rails per importare file `.yml` locali.
- Il task deve leggere YAML, convertirlo in hash/JSON e fare upsert nel DB usando `name`.
- Scopo: popolare dati provvisori mentre brand, percorsi, arcipelaghi e contenuti sono ancora in costruzione.
- Non creare ancora UI admin: prima serve una base dati comoda e modificabile velocemente.
