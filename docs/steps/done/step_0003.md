# Step 0006 - Pagine pubbliche e asset

## Stato attuale

Le pagine pubbliche principali vivono in:

```text
app/views/pages/flowpulse.html.erb
app/views/pages/markpostura.html.erb
app/views/pages/posturacorretta.html.erb
```

Sono raggiungibili sia da route Rails dirette sia da domini dedicati tramite `Domain`.

```text

/posturacorretta
```

Domini:

```text
markpostura.it       -> pages#markpostura
posturacorretta.org  -> pages#posturacorretta
flowpulse.net        -> pages#flowpulse
```

app/views/pages/markpostura.html.erb

  - `Il Giardino del Corpo`: da aggiornare;
  - `Spontaneamente`: coming soon;
  - `Il corpo, un mondo da scoprire`: coming soon.

Asset ImageKit usati:

```text
https://ik.imagekit.io/posturacorretta/markpostura/home/
```

Regola:

- usare ImageKit per le immagini pubbliche pesanti;
- evitare di caricare immagini da Pinterest come hotlink diretto;
- Pinterest puo essere ispirazione o archivio, ma la pagina deve usare asset stabili.

## PosturaCorretta

File:

```text
app/views/pages/posturacorretta.html.erb
```

Stato:

- hero introduttiva;
- sezione `#percorso` sul rapporto tra diagnosi e percorso;
- CTA verso `#inizia`;
- scelta guidata a tab/card:
  - rosso;
  - verde;
  - blu;
  - viola;
- contenuto percorso generato via JavaScript;
- CTA `Inizia il percorso` in alto e in fondo alla lista tappe;
- CTA attualmente disabilitate finche non esiste il flusso reale;
- box professionisti/aderisci sotto il percorso, non piu nell'aside;
- box professionisti cambia testo e colore in base alla tab selezionata.

Decisione UX:

- le tappe del percorso restano informative;
- non usare tanti link `Approfondisci` sulle singole tappe;
- usare una CTA principale ripetuta in due punti:
  - sopra le tappe;
  - dopo le tappe.

## Da fare

- Decidere la destinazione reale di `Inizia il percorso`.
- Quando esiste la destinazione, trasformare le CTA disabilitate in link/bottoni attivi.
- Valutare se estrarre i dati delle tab PosturaCorretta da HTML inline a JSON/YAML o DB.
- Valutare se `markpostura` e `posturacorretta` devono condividere componenti/partial.
- Pulire asset locali pesanti non piu usati, dopo conferma che ImageKit e definitivo.

## Non fare ora

- Non creare un controller dedicato per ogni dominio.
- Non rendere cliccabili le tappe se le pagine reali non esistono.
- Non introdurre una UI admin per questi contenuti prima di stabilizzare struttura e testi.
